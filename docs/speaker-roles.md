# Azure OpenAI Research: Agent/Caller Role Identification

## Նպատակ

Այս հետազոտության նպատակն է ուսումնասիրել **Azure OpenAI** ծառայության հնարավորությունները՝ արդեն տրանսկրիպտացված (transcribed) հաճախորդների սպասարկման խոսակցություններում ավտոմատ կերպով որոշելու յուրաքանչյուր հաղորդագրության խոսողի դերը։

Համակարգը պետք է կարողանա մշակել տարբեր լեզուներով և խոսակցական ոճերով ներկայացված երկխոսություններ՝ պահպանելով խոսակցության հաջորդականությունը և յուրաքանչյուր հաղորդագրությանը վերագրելով համապատասխան դեր։

Հիմնական դերերը հետևյալն են.

- **Agent** – հաճախորդների սպասարկման ներկայացուցիչ
- **Caller** – հաճախորդ կամ ծառայությունից օգտվող անձ

---

# Հետազոտության նպատակները

Հետազոտության ընթացքում ուսումնասիրվում են հետևյալ հնարավորությունները.

- Azure OpenAI-ի կիրառումը խոսողների դերերի ավտոմատ որոշման համար։
- Տարբեր լեզուներով և բարբառներով տրանսկրիպտների մշակման հնարավորությունները։
- GPT մոդելների կարողությունը հասկանալու խոսակցության համատեքստը և դրան համապատասխան որոշելու յուրաքանչյուր հաղորդագրության խոսողին։
- JSON ձևաչափով կառուցվածքային արդյունքի ստացումը։

---

# Կատարված աշխատանք

## 1. Azure OpenAI Resource-ի ստեղծում

Azure Portal-ում ստեղծվել է **Azure OpenAI Resource**, որը նախատեսված է GPT մոդելների հետ աշխատելու համար։

Ստեղծման ընթացքում իրականացվել են հետևյալ քայլերը.

- Azure Subscription ընտրություն
- Resource Group ստեղծում
- Region ընտրություն
- Azure OpenAI Resource-ի ստեղծում

---

## 2. Azure AI Foundry-ի օգտագործում

Azure OpenAI Resource-ի ստեղծումից հետո օգտագործվել է **Azure AI Foundry** միջավայրը։

Azure AI Foundry-ն օգտագործվել է հետևյալ նպատակներով.

- GPT մոդելների ընտրություն և փորձարկում (Օգտագործվել է gpt 5 mini մոդելը)
- Prompt-երի մշակում
- Model Deployment-ի կառավարում
- API Endpoint-ների ստացում
- Մոդելի պատասխանների փորձարկում առանց ծրագրային կոդ գրելու

---

## 3. GPT System Prompt-ի մշակում

Ստեղծվել է հատուկ **System Prompt**, որի նպատակն է GPT մոդելին տրամադրել հստակ հրահանգներ խոսողների դերերը որոշելու համար։

Prompt-ը նախատեսված է հետևյալ տվյալների մշակման համար.

- Անգլերեն լեզու
- Հայերեն լեզու
- Արևելահայերեն
- Արևմտահայերեն
- Տարածաշրջանային հայերեն բարբառներ (օրինակ՝ Երևանի, Գյումրու, Լոռու, Սյունիքի և այլ տարածաշրջանների խոսվածքներ)
- Խոսակցական հայերեն
- Սլենգ և ոչ ֆորմալ արտահայտություններ
- Խառը լեզուներ (Armenian + English + Russian)
- Լատինատառ հայերեն (Transliteration)
- Փոքր ուղղագրական կամ քերականական սխալներով տեքստեր

Prompt-ը նախատեսված է նաև այն դեպքերի համար, երբ տրանսկրիպտում բացակայում են **Agent** և **Caller** պիտակները։


# Prompt-ի հիմնական հնարավորությունները

Ստեղծված Prompt-ը նախատեսված է.

- Agent/Caller դերերի ավտոմատ որոշման համար
- Խոսակցության համատեքստի վերլուծության համար
- Հարց–պատասխան կառուցվածքի ճանաչման համար
- Բազմալեզու խոսակցությունների մշակման համար
- Հայերենի տարբեր բարբառների և խոսակցական ձևերի ընկալման համար
- Կարճ և երկիմաստ պատասխանների («Այո», «Հա», «Okay», «Sure», «Չէ») ճիշտ մեկնաբանման համար
- Տեքստի բնօրինակ ձևի պահպանման համար՝ առանց թարգմանելու կամ փոփոխելու
- JSON ձևաչափով կառուցվածքային արդյունքի վերադարձման համար

# Փորձարկման պլան

Հետազոտության ընթացքում նախատեսվում է փորձարկել հետևյալ սցենարները.

- Անգլերեն հաճախորդների սպասարկման խոսակցություններ
- Գրական հայերենով խոսակցություններ
- Արևելահայերեն
- Արևմտահայերեն
- Տարածաշրջանային բարբառներ (օրինակ՝ Երևանի, Գյումրու և այլ տարածաշրջանների խոսվածքներ)
- Խոսակցական հայերեն
- Խառը լեզուներով (Armenian + English + Russian) խոսակցություններ
- Լատինատառ հայերենով գրված հաղորդագրություններ
- Կարճ և երկիմաստ պատասխաններ
- Խոսակցություններ առանց Agent/Caller պիտակների

# Սահմանափակումներ

Azure OpenAI-ն խոսողների դերերը որոշում է միայն տեքստի և խոսակցության համատեքստի հիման վրա։

Եթե տրանսկրիպտը չափազանց կարճ է, չունի բավարար համատեքստ կամ բաղկացած է միայն կարճ պատասխաններից, հնարավոր է դերերի որոշման ճշգրտությունը նվազի։

Բարբառների և խոսակցական արտահայտությունների ճանաչման որակը կախված է GPT մոդելի լեզվական գիտելիքներից և օգտագործվող մոդելի տարբերակից։

# Հաջորդ քայլերը

- Azure OpenAI-ի միջոցով իրականացնել փորձնական հարցումներ
- Փորձարկել տարբեր GPT մոդելներ
- Գնահատել Agent/Caller դասակարգման ճշգրտությունը
- Փորձարկել անգլերեն, հայերեն և տարածաշրջանային բարբառներով տրանսկրիպտներ
- Վերլուծել ստացված արդյունքները
- Փաստաթղթավորել բոլոր փորձարկումները և եզրակացությունները

# Եզրակացություն

Նախնական հետազոտությունը ցույց է տալիս, որ Azure OpenAI GPT մոդելները կարող են արդյունավետ կիրառվել արդեն տրանսկրիպտացված հաճախորդների սպասարկման խոսակցություններում **Agent** և **Caller** դերերի ավտոմատ որոշման համար։

Azure OpenAI-ն օգտագործում է մեծ լեզվային մոդելներ (Large Language Models), որոնք կարող են հասկանալ ոչ միայն անգլերեն և գրական հայերեն, այլ նաև հայերենի տարբեր բարբառներ, խոսակցական արտահայտություններ, սլենգ և խառը լեզուներով կազմված նախադասություններ։

Հետագա փուլում նախատեսվում է իրականացնել գործնական փորձարկումներ տարբեր լեզուներով, բարբառներով և իրական տրանսկրիպտներով՝ գնահատելու մոդելի ճշգրտությունը, սահմանափակումները և կիրառելիությունը։
